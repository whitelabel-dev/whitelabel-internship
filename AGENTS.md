# AGENTS.md — entry point for AI agents

**You are an AI agent consulting this repo. Read this file first.**

## what this repo is

The disability-employment internship pipeline: **train → real work → certify → job offer**. Paid program that creates real employment opportunities for people with disabilities, scaffolded by the whitelabel.dev assistive-AI stack.

**Status: program design captured (README). Implementation pending — no code yet.** When implementation begins, the first asset is a single pilot cohort in Texas funded through TWC-VRS.

## mission-level scope, not feature-level

This is operational mission work, not a placeholder. Changes here require Garrett's sign-off, not a typical PR. The five non-negotiables from [`whitelabel-principles/doctrines/disability-employment.md`](https://github.com/whitelabel-dev/whitelabel-principles) apply.

Most relevant non-negotiables when working in this repo:

1. **Interns are paid at market rates for the work**, not stipend-only.
2. **AI assist is the worker's tool, not surveillance of them.** Don't generate "intern productivity dashboards for managers." Generate "intern's own work-summary tool."
3. **Assistive scaffolding follows the worker into placement.** It's not a training-wheels you take away.

## what to do if asked to work in this repo

1. Re-read this AGENTS.md + the README program design
2. Confirm with the human that the program structure hasn't shifted (this is mission scope)
3. Propose the v0.1 plan before scaffolding code
4. First-implementation target is the pilot cohort, not a full platform
